Kimi K3 (max thinking)  vs.  the July frontier

Twenty-three real merged PRs across Python, Rust, TypeScript, JavaScript, and Go, graded by deterministic hidden tests
Moonshot’s 2.8T open-weight flagship lands at 17/23 (74%) — and every one of its misses is a 30-minute timeout, not a wrong patch. At 18.1 min/task it is 4–5× slower than the field, and Haiku 4.5 outsolves it (18/23) at half the cost per solved task.

1. The clock is the failure mode. All five K3 zero-scores ran out the 30-minute task budget mid-work; it never submitted a wrong final patch. One partial credit (SQLGlot ISO-8601, 0.7).

2. Always-on max thinking is slow. 18.1 min/task average against 3.4–5.1 for the field — with moderate tokens (134 K/task), the cost is time, not volume.

3. A small closed model beats the open flagship here. Haiku 4.5: one more solve, half the $/solved, 3.5× the speed — on time-budgeted agentic work, latency is accuracy.

4. Integration is part of the score. Three K3 solves were initially lost to 120-second HTTP read timeouts on judge calls; a 900 s provider timeout recovered all three.

5. Given time, it closes the gap. Uncapped ablation (2 h wall clock, 400 steps, $4/run): aiohttp, NetworkX-Leiden, and jiff-strftime all fall — 20/23 (87%), level with Fable 5 and Sol. The two remaining partials are the tasks no model has ever solved.

Method. VulcanBench v3: 23 tasks from real merged post-cutoff PRs (jiff, itertools, zod, hono, node-semver, cobra, pflag, chi, flask, aiohttp, sqlglot, packaging, more-itertools, networkx, pennylane), network-isolated Docker sandboxes, deterministic hidden-test grading, one attempt per task. kimi-k3 via the Moonshot API ($3/$15 per M tokens); thinking is always on and reasoning_effort supports only “max”, so a single configuration is reported. Kimi suite spend $16.84 under a $40 cap; the finding-5 ablation re-ran only the five timed-out tasks with --override-budgets (runs marked budget_override, excluded from Table 1 and Figure 1; +$17.05). † Fable 5 ran with Opus 4.8 refusal fallback (see Report No. 07).
