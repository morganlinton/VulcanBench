When a command has an OPTIONAL argument whose type enumerates its choices (for example `click.Choice(["a", "b", "c"])`), the usage synopsis wraps the choices in a second pair of square brackets — it shows `[[a|b|c]]` instead of `[a|b|c]`. Required arguments, and optional arguments whose type does not already bracket its metavar, are not affected. Fix the synopsis so an optional choice-style argument shows a single pair of brackets.
