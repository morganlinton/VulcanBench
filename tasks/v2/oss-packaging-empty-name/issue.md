Parsing a wheel or sdist filename whose project-name component is empty (for example a filename that begins with the version, so the name part before it is blank) currently succeeds and returns an empty project name. Such filenames are invalid and should be rejected with the appropriate error (`InvalidWheelFilename` for wheels, `InvalidSdistFilename` for sdists). Fix the filename parsers so an empty project name is rejected, while valid filenames continue to parse as before.
