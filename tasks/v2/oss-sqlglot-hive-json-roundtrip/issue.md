In the Hive dialect, the JSON functions `PARSE_JSON` and `TO_JSON` do not round-trip. Reading and re-generating `SELECT PARSE_JSON('{}')` in Hive drops the `PARSE_JSON` call (producing `SELECT '{}'`), and `SELECT TO_JSON(PARSE_JSON('{}'))` is rewritten into a broken expression instead of being preserved. These functions should round-trip unchanged when reading and writing Hive SQL. Fix the Hive generation so `PARSE_JSON` and `TO_JSON` are preserved correctly.
