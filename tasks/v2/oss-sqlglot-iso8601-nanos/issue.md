Transpiling Presto SQL that calls `FROM_ISO8601_TIMESTAMP_NANOS('<iso timestamp string>')` to dialects that do not have that function (Snowflake, Spark, Databricks, BigQuery, DuckDB) currently emits the function call unchanged, producing SQL those engines cannot run. It should instead be converted to an equivalent timestamp cast for each dialect (a timestamp-with-time-zone cast where the target dialect defaults to that, a plain timestamp cast otherwise). Reading and writing Presto itself must keep the function as-is. Fix the transpilation so a Presto `FROM_ISO8601_TIMESTAMP_NANOS(...)` call produces the correct cast when written to the other dialects.
