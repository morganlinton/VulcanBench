When SQL contains a parenthesized expression as the first element of a multi-element comma-separated list, for example `((SELECT 42), 1)`, sqlglot's parser incorrectly treats the parenthesized part as a Subquery and drops the remaining elements, so `((SELECT 42), 1)` parses as just `((SELECT 42))` instead of a two-element tuple. When a multi-expression comma list is present, a parenthesized expression should be parsed as a Tuple. Fix the parser so `((SELECT 42), 1)` parses as a Tuple containing both elements, while plain tuples such as `(1, 2)` remain unaffected.
